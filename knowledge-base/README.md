# دانش‌نامه‌ی چت‌بات آرکان (RAG)

این فولدر مجموعه‌ی نمونه‌ی اسناد شرکت آرکان است که برای تغذیه‌ی چت‌بات به‌سبک **RAG** (تولید مبتنی بر بازیابی) آماده شده است. هدف این است که نشان دهد یک شرکت معمولاً چه نوع داده‌هایی و در چه فرمت‌هایی به چت‌بات می‌دهد تا بتواند به سؤالات مشتریان پاسخ دقیق و هماهنگ با برند بدهد.

> همه‌ی محتوا فارسی و هماهنگ با لحن برند آرکان است. نام فایل‌ها و فولدرها انگلیسی است تا با ابزارهای پردازش بهتر کار کند.

## ساختار فولدرها

| فولدر | محتوا |
|-------|-------|
| `01-company/` | معرفی شرکت، حقایق کلیدی، تیم |
| `02-services/` | کاتالوگ خدمات، شرح خدمات، متدولوژی، قیمت‌گذاری |
| `03-process/` | فرایند همکاری و چک‌لیست شروع |
| `04-support/` | پرسش‌های متداول، پاسخ به نگرانی‌ها، تماس |
| `05-proof/` | مطالعات موردی و نظرات مشتریان |
| `06-content/` | مقالات بینش و واژه‌نامه |
| `07-policies/` | سیاست‌ها و بروشور شرکت |

## پوشش فرمت‌ها

این مجموعه عمداً فرمت‌های متنوعی دارد تا یک خط لوله‌ی RAG واقعی را شبیه‌سازی کند:

| فرمت | نمونه فایل | کاربرد در RAG |
|------|-----------|----------------|
| Markdown (`.md`) | `company-profile.md`، مقالات | متن غنی و ساختاریافته؛ مناسب‌ترین برای chunking |
| JSON (`.json`) | `faq.json`، `services-catalog.json` | داده‌ی ساختاریافته‌ی پرسش/پاسخ و کاتالوگ |
| YAML (`.yaml`) | `company-facts.yaml` | حقایق کلیدی برای پاسخ‌های کوتاه و دقیق |
| CSV (`.csv`) | `pricing-packages.csv`، `glossary.csv` | داده‌ی جدولی (قیمت، واژه‌نامه) |
| TXT (`.txt`) | `contact-and-hours.txt` | متن ساده و سریع |
| HTML (`.html`) | `company-brochure.html` | سند صفحه‌بندی‌شده (منبع PDF) |
| PDF (`.pdf`) | `company-brochure.pdf` | سند نهایی برای دانلود/ایندکس |
| DOCX (`.docx`) | `company-profile.docx` | سند آفیس؛ نمونه‌ی ورودی رایج RAG |

## فهرست اسناد
فایل [`manifest.json`](manifest.json) فهرست کامل تمام اسناد را با متادیتا (عنوان، مسیر، فرمت، موضوع، مخاطب) دارد. این فایل نقطه‌ی ورود مناسبی برای اسکریپت تزریق (ingestion) است.

## نکته‌ی مهم درباره‌ی داده‌ی نمونه
- **اطلاعات واقعی شرکت** (نام، تأسیس ۱۳۹۶، تماس، خدمات، متدولوژی، ارزش‌ها، نقل‌قول‌ها) از اسناد رسمی برند آرکان استخراج شده است.
- **قیمت‌ها، مطالعات موردی، اعضای تیم (به‌جز بنیان‌گذار) و برخی نظرات** به‌عنوان **نمونه‌ی واقع‌گرایانه** ساخته شده‌اند، چون در منابع موجود نبودند. این موارد در سرآیند فایل‌های مربوطه علامت‌گذاری شده‌اند.

## مرحله‌ی بعد (خارج از محدوده‌ی این فولدر)
این فولدر فقط **داده‌ی نمونه** است. برای راه‌اندازی RAG واقعی، گام بعدی ساخت یک خط لوله است که این اسناد را می‌خواند، آن‌ها را به قطعه (chunk) می‌شکند، embedding می‌سازد و در یک پایگاه برداری (مثلاً `pgvector` روی همان Supabase پروژه) ذخیره می‌کند.
