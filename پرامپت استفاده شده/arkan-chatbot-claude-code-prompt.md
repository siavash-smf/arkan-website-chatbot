# پرامپت ساخت چت‌بات آرکان (RAG + چندکاناله) — برای Claude Code

> این فایل را همراه `arkan-client-brief.md` و `arkan-brand-guide.md` در ریشه‌ی پروژه قرار بده و متن زیر را به Claude Code بده. این چت‌بات روی همان پروژه‌ی Next.js و همان Supabaseِ وب‌سایت آرکان سوار می‌شود.

---

## نقش و هدف

تو یک مهندس ارشد فول‌استک و متخصص سیستم‌های RAG هستی. قرار است یک **چت‌بات هوشمند با معماری RAG** برای شرکت «آرکان» بسازی. این یک نمونه‌ی آموزشی کامل و حرفه‌ای است؛ پس کد باید تمیز، ماژولار و قابل‌توضیح باشد.

**پیش از شروع:**
1. اسکیل `/ui-ux-pro-max` را برای تمام بخش‌های UI (ویجت، صفحه‌ی چت، پنل مدیریت) فعال کن.
2. دو فایل `arkan-client-brief.md` و `arkan-brand-guide.md` را بخوان. شخصیت، لحن و رنگ‌های چت‌بات باید کاملاً با برند آرکان هم‌خوان باشد.

**فلسفه‌ی معماری: «یک مغز، چند کانال».** یک هسته‌ی مرکزی تمام منطق گفتگو، RAG و تماس با مدل را مدیریت می‌کند؛ کانال‌ها (تلگرام، ویجت، صفحه‌ی کامل) فقط لایه‌ی ورودی/نمایش‌اند و همگی از همان مغز استفاده می‌کنند.

---

## استک فنی

- **Next.js 14+** (App Router) + **TypeScript** — همان پروژه‌ی سایت آرکان
- **Supabase** + افزونه‌ی **pgvector** برای ذخیره و جست‌وجوی برداری — همان دیتابیس سایت
- **تولید پاسخ از طریق OpenRouter (یک کلید برای همه‌ی مدل‌ها):** همه‌ی مدل‌های تولید پاسخ از طریق **OpenRouter** فراخوانی می‌شوند. یک کلید API از OpenRouter گرفته می‌شود و همه‌ی مدل‌ها (Claude، Gemini، GPT، Qwen و…) با همان یک کلید و base URL واحد (`https://openrouter.ai/api/v1`) و فرمت سازگار با OpenAI در دسترس‌اند. تعویض مدل فقط تغییر یک رشته (model slug) است، نه تغییر کلید یا کد. پاسخ‌ها **استریمی** باشند. شناسه‌ی مدل‌ها به فرمت namespaced خود OpenRouter است (مثل `anthropic/...`، `google/...`، `openai/...`، `qwen/...`)؛ slug دقیق هر مدل از کاتالوگ OpenRouter گرفته شود.
- **Embedding** قابل‌تنظیم و چندارائه‌دهنده: پیش‌فرض یک مدل چندزبانه‌ی قوی. توجه: OpenRouter برای مدل‌های چت است، نه embedding؛ پس **کلید embedding جداگانه** (Cohere / OpenAI / Google / Voyage) تنظیم می‌شود. برای محتوای **فارسی**، کیفیت چندزبانه‌ی مدل embedding تعیین‌کننده‌ی کیفیت کل سیستم است. (لیست مدل‌ها در بخش پنل مدیریت.)
- **Telegram Bot API** با Webhook
- **Tailwind CSS** + RTL کامل، فونت‌های استعداد/وزیرمتن
- آماده‌ی استقرار روی Vercel (و یک سرویس برای webhook تلگرام)

## بخش اول: مغز مرکزی (Core Engine)

این لایه باید مستقل از کانال‌ها و در قالب سرویس/ماژول‌های قابل‌استفاده‌ی مشترک نوشته شود.

### ۱. پایپ‌لاین RAG
- **Ingestion:** دریافت منبع (PDF / Word / متن / URL برای کرال) → استخراج متن → پاک‌سازی.
- **Chunking:** قطعه‌بندی متن با اندازه و overlap **قابل‌تنظیم** (پیش‌فرض ~۵۰۰ توکن، overlap ~۵۰).
- **Embedding:** تبدیل هر chunk به بردار با provider/مدل **قابل‌تنظیم از پنل** و ذخیره در pgvector.
- **Retrieval:** embed کردن سؤال کاربر (با `input_type=query`) → جست‌وجوی شباهت (top_k قابل‌تنظیم) → فیلتر بر اساس آستانه‌ی شباهت → (اختیاری) reranker.
- **Generation:** سرهم‌کردن context بازیابی‌شده + system prompt + تاریخچه‌ی گفتگو → تماس استریمی با Claude → بازگرداندن پاسخ به‌همراه **منابع (citations)**.

### ۲. مدیریت گفتگو و حافظه
- ذخیره‌ی هر پیام و session؛ پشتیبانی از گفتگوی چندمرحله‌ای.
- خلاصه‌سازی خودکار گفتگوهای طولانی برای کنترل طول context.

### ۳. System Prompt و پرسونا
- از دیتابیس خوانده شود (نه hardcode) تا از پنل قابل تغییر باشد.
- لحن و محدوده طبق برند گاید آرکان: حرفه‌ای، اطمینان‌بخش بدون اغراق، خطاب «شما». بات نباید مشاوره‌ی تخصصی قطعی بدهد؛ هدفش راهنمایی و هدایت کاربر به **ثبت درخواست مشاوره** است.

### ۴. ابزارها (Function Calling)
- **ثبت لید / رزرو مشاوره:** وقتی کاربر آماده است، بات اطلاعات را می‌گیرد و در جدول `leads` ثبت می‌کند (یکپارچه با فرم سایت).
- **تحویل به انسان (handoff):** وقتی سؤال خارج از حوزه است یا کاربر درخواست می‌کند.

### ۵. گاردریل
- محدودکردن پاسخ به حوزه‌ی آرکان، مدیریت سؤالات نامرتبط، و عدم ارائه‌ی وعده‌ی تضمینی.

## بخش دوم: مدل داده (Supabase)

این جداول را بساز (با RLS مناسب):

```
documents        (id, title, source_type, source_url, status, tags, created_at)
chunks           (id, document_id, content, embedding vector, token_count, chunk_index, metadata)
conversations    (id, channel, external_user_id, status, summary, started_at)
messages         (id, conversation_id, role, content, model_used, tokens_in, tokens_out, retrieved_chunk_ids, created_at)
unified_users    (id, channel, external_id, name, first_seen)
leads            (… همان جدول سایت + source, conversation_id …)
feedback         (id, message_id, rating, comment, created_at)
prompt_versions  (id, content, persona, is_active, created_at, created_by)
model_config     (id, channel, provider, active_model, temperature, max_tokens, top_p, fallback_provider, fallback_model, schedule, updated_at)
embedding_config (id, provider, model, dimensions, chunk_size, chunk_overlap, top_k, similarity_threshold, reranker_enabled, reranker_model)
admin_users      (id, email, role, created_at)
audit_log        (id, admin_user_id, action, target, created_at)
```

## بخش سوم: کانال‌ها

همگی از همان مغز مرکزی استفاده می‌کنند.

### ۱. تلگرام
- اتصال با Webhook؛ مدیریت دستورهای `/start`، `/help`، `/reset`.
- دکمه‌های inline برای quick replies.
- مدیریت rate limit برای هر کاربر.
- پشتیبانی از **broadcast** (ارسال پیام انبوه از پنل).
- نگاشت `chat_id` به جدول `unified_users`.

### ۲. ویجت وب (Embeddable Widget)
- یک حباب شناور گوشه‌ی صفحه که با یک اسنیپت `<script>` در هر سایتی قابل نصب باشد.
- ظاهر (رنگ، موقعیت، پیام خوش‌آمد) از پنل قابل شخصی‌سازی.
- محدودسازی به **دامنه‌های مجاز**.
- استریم پاسخ، typing indicator، بازخورد 👍/👎.

### ۳. صفحه‌ی کامل چت (در سایت آرکان)
- یک مسیر اختصاصی (مثلاً `/consultant`) با تجربه‌ی تمام‌صفحه.
- سؤالات پیشنهادی شروع، تاریخچه‌ی session، نمایش منابع پاسخ، و دکمه‌ی «درخواست مشاوره».
- برندینگ و RTL کامل طبق برند گاید.

## بخش چهارم: پنل مدیریت چت‌بات

به پنل مدیریت موجود سایت اضافه شود. این بخش باید **کامل و حرفه‌ای** باشد و شامل تمام موارد زیر:

**۱. داشبورد و تحلیل** — تعداد گفتگوها/کاربران یکتا/پیام‌ها، تفکیک کانال، نرخ تبدیل به لید، میانگین زمان و طول گفتگو، نرخ رضایت، پرتکرارترین موضوعات، و **مصرف توکن و هزینه به‌تفکیک مدل**.

**۲. مدیریت پایگاه دانش (RAG)** — آپلود منابع (PDF/Word/متن/URL)، مشاهده/ویرایش/حذف، نمایش وضعیت پردازش، re-index دستی، و جست‌وجوی آزمایشی در KB.

**۳. تنظیمات Embedding** — انتخاب provider و مدل، dimensions، chunk_size و overlap، استراتژی chunking، و input_type. هنگام تغییر مدل، **هشدار نیاز به بازسازی کل ایندکس** نمایش داده شود. به‌علاوه تنظیمات retrieval: top_k، similarity_threshold، و فعال/غیرفعال‌سازی reranker.

مدل‌های embedding که باید در منوی انتخاب باشند (همگی چندزبانه و مناسب فارسی):
- **Cohere** (`embed-multilingual-v3.0`) — تمرکز ویژه‌ی Cohere روی مدل‌های چندزبانه؛ نسخه‌ی v3 یکی از قوی‌ترین گزینه‌ها برای فارسی و عملکرد فوق‌العاده در سیستم‌های RAG و درک معنای جملات.
- **OpenAI** (`text-embedding-3-large` و `text-embedding-3-small`) — نسل سوم OpenAI با پشتیبانی بسیار بهتر از زبان‌های غیرانگلیسی از جمله فارسی. نسخه‌ی Small مقرون‌به‌صرفه و سریع، نسخه‌ی Large بالاترین دقت با قابلیت تغییر ابعاد خروجی (dimensions).
- **Google** (`text-multilingual-embedding`) — به‌دلیل آموزش روی دیتاست‌های عظیم وب، درک بسیار خوبی از متون فارسی و جست‌وجوی معنایی دارد و دسترسی از طریق API ساده است.
- **Voyage AI** (`voyage-multilingual-2`) — استارتاپ متخصص embedding (ارائه‌دهنده‌ی ترجیحی Anthropic)؛ نسخه‌ی چندزبانه‌اش در بنچمارک‌ها در فارسی رقابت بسیار نزدیکی با OpenAI و Cohere دارد.

> توجه: هر مدل dimensions متفاوتی دارد؛ ستون `embedding` در pgvector باید با مدل فعال هماهنگ باشد و تغییر مدل نیاز به re-embed کل پایگاه دانش دارد.

**۴. انتخاب و تنظیم مدل هوش مصنوعی** — منوی انتخاب مدل تولید پاسخ باید **چندارائه‌دهنده** باشد. مدل‌ها بر اساس ارائه‌دهنده دسته‌بندی شوند:

*Anthropic (Claude):*
- `claude-haiku-4-5` (Haiku 4.5 — سریع، کم‌هزینه، لحن بسیار مودب و انسانی برای پشتیبانی مشتری؛ جانشین به‌روز Claude 3 Haiku)

*Google (Gemini):*
- `gemini-3-flash` / `gemini-3.1-flash-lite` (سریع، بسیار ارزان، Context Window بسیار بزرگ و عالی برای RAG؛ جانشین‌های به‌روز Gemini 2.5 Flash)
- `gemini-2.5-flash` (هنوز در دسترس)

*OpenAI:*
- `gpt-5-mini` / `gpt-5.4-nano` (ارزان و سریع، درک خوب اصطلاحات فارسی و لحن طبیعی؛ جانشین‌های به‌روز GPT-4o-mini)
- `gpt-4o-mini` (برای سازگاری، اگر لازم شد)

*گزینه‌ی ارزان/چندزبانه‌ی دیگر:*
- `qwen-3.5` (علی‌بابا — چندزبانه‌ی قوی، بسیار ارزان، open-weight و مناسب میزبانی نزدیک به کاربر ایرانی)

> شناسه‌ها بالا برای خوانایی‌اند؛ در کد، slug رسمی هر مدل از کاتالوگ OpenRouter استفاده شود (مثل `anthropic/claude-haiku-4.5`).

با امکان: تعیین مدل پیش‌فرض، **زمان‌بندی تغییر مدل بر اساس روز/بازه** (تا در روزهای مختلف مدل عوض شود)، تخصیص مدل متفاوت به هر کانال، تعریف مدل **fallback** (مثلاً وقتی مدل اصلی خطا/rate limit داد)، و تنظیم پارامترها (temperature، max_tokens، top_p) و سقف هزینه. چون همه‌ی مدل‌ها از طریق OpenRouter می‌آیند، تعویض مدل فقط تغییر یک slug است.

**۵. پرسونا و System Prompt** — ویرایشگر با **نسخه‌بندی و بازگشت (versioning)**، تنظیم لحن/محدوده طبق برند، قوانین رفتاری، و پیام خوش‌آمد و quick replies هر کانال.

**۶. مدیریت گفتگوها (Inbox)** — مشاهده‌ی همه‌ی مکالمات با فیلتر و جست‌وجو، و نمایش **منابع RAG هر پاسخ** برای دیباگ.

**۷. تحویل به اپراتور انسانی** — صف نیازمند انسان، اعلان، ورود زنده و توقف موقت بات، بازگشت به بات.

**۸. مدیریت لید** — لیدهای چت‌بات یکپارچه با جدول `leads` سایت، با ثبت منبع و گفتگوی مرتبط، وضعیت پیگیری و خروجی Excel/CSV.

**۹. بازخورد و بهبود** — مشاهده‌ی 👎ها و گفتگوهای پرچم‌خورده، و لیست **سؤالات بی‌جواب** با پیشنهاد افزودن به KB.

**۱۰. کانال‌ها و یکپارچه‌سازی** — تنظیم توکن و webhook تلگرام، تولید و شخصی‌سازی کد embed ویجت + دامنه‌های مجاز، و ابزار **broadcast تلگرام**.

**۱۱. پلی‌گراند و تست** — تست تغییرات قبل از اعمال زنده، مقایسه‌ی پاسخ دو مدل کنار هم، و نمایش chunkهای بازیابی‌شده برای هر کوئری.

**۱۲. کاربران پنل و دسترسی‌ها** — نقش‌ها (مالک/ادمین، ویرایشگر محتوا، اپراتور، فقط‌خواندنی) و audit log.

**۱۳. تنظیمات و امنیت** — مدیریت امن کلیدهای API، rate limiting و آنتی‌اسپم، تنظیمات data retention و حریم خصوصی، و هشدار سقف بودجه.

## ترتیب کاری که از تو می‌خواهم

1. اسکیل `/ui-ux-pro-max` را فعال و دو فایل brief و brand-guide را بخوان؛ خلاصه‌ی درکت را بده.
