# پرامپت ساخت وب‌سایت آرکان — برای Claude Code

> این فایل را همراه `arkan-client-brief.md` و `arkan-brand-guide.md` در ریشه‌ی پروژه قرار بده و متن زیر را به Claude Code بده.

---

## نقش و هدف

تو یک مهندس فرانت‌اند و طراح ارشد هستی. قرار است وب‌سایت رسمی شرکت **«آرکان»** (مشاور استراتژی و رشد کسب‌وکار) را بسازی. قبل از هر کاری، دو فایل `arkan-client-brief.md` و `arkan-brand-guide.md` را در ریشه‌ی پروژه بخوان و **تمام** تصمیم‌های محتوایی و بصری را بر اساس آن‌ها بگیر.

**مهم:** برای طراحی و ساخت کل وب‌سایت از اسکیل `/ui-ux-pro-max` استفاده کن. این اسکیل را در ابتدای کار فعال کن و اصول طراحی و کیفیت بصری آن را در تمام بخش‌ها رعایت کن. توکن‌ها و قواعد برند گاید آرکان باید در چارچوب همین اسکیل اعمال شوند.

**هدف اصلی سایت یک چیز است:** بازدیدکننده را به پر کردن «فرم درخواست مشاوره» برساند. سایت فروش آنلاین یا تراکنش ندارد.

---

## استک فنی

- **Next.js 14+** با App Router و **TypeScript**
- **Tailwind CSS** برای استایل
- **RTL** کامل (فارسی) — `dir="rtl"` و `lang="fa"` روی `<html>`
- فونت‌ها: **استعداد (Estedad)** برای عناوین و **وزیرمتن (Vazirmatn)** برای متن، از طریق `@font-face` با CDN (jsDelivr). اگر ممکن بود از `next/font/local` استفاده کن.
- فرم: ذخیره در **Supabase** (جدول `leads`) + ارسال ایمیل اطلاع‌رسانی. اگر Supabase در دسترس نبود، یک API route با ذخیره‌ی موقت بساز و نقطه‌ی اتصال را کامنت‌گذاری کن.
- آماده‌ی استقرار روی **Vercel**.

> این استک عمداً با فاز بعدی (چت‌بات) هم‌خوان است؛ پایگاه داده‌ی Supabase مشترک خواهد بود.

## توکن‌های طراحی (از برند گاید)

در `tailwind.config` این رنگ‌ها را تعریف کن:

```
pine:  #143A32   (اصلی)
brass: #B5853A   (تأکید)
bone:  #F7F3EC   (پس‌زمینه‌ی روشن)
sand:  #E7DECF   (سطوح ثانویه)
ink:   #15201C   (متن اصلی)
slate: #5A5F5B   (متن فرعی)
```

- شعاع گوشه: کارت `12px`، دکمه `8px`
- فضای سفید زیاد، سایه‌های بسیار ملایم، بدون گرادیان پررنگ
- نسبت رنگ: ۶۰٪ روشن، ۳۰٪ سبز کاج، ۱۰٪ برنجی
- **CTA اصلی:** پس‌زمینه‌ی pine + متن bone (کنتراست بالا). برنجی را برای متن کوچک روی سفید استفاده نکن.

مقیاس تایپ: H1 ۴۸px، H2 ۳۲px، H3 ۲۴px، بدنه ۱۷px با line-height ۱.۸.

## ساختار صفحه (تک‌صفحه‌ای با اسکرول + لینک‌های لنگر)

سایت یک صفحه‌ی اصلی با این بخش‌ها به‌ترتیب:

1. **هدر چسبان (Sticky Header)**
   - لوگوی متنی «آرکان» (راست)، منوی لنگری (خدمات، فرایند، درباره‌ی ما، تماس)، و یک دکمه‌ی CTA «درخواست مشاوره» (چپ) که به فرم اسکرول می‌کند.

2. **هیرو (Hero)**
   - تیتر اصلی قدرتمند درباره‌ی رشد پایدار کسب‌وکار (با لحن برند گاید بنویس).
   - یک زیرعنوان کوتاه و توضیحی.
   - دکمه‌ی اصلی «درخواست مشاوره‌ی رایگان» → اسکرول به فرم.
   - فضای سفید زیاد، بدون شلوغی.

3. **خدمات (Services)**
   - پنج خدمت از brief به‌صورت کارت (با آیکون خطی ساده و یک‌خط توضیح هرکدام).

4. **متدولوژی چهار رکن**
   - چهار رکن (استراتژی، ساختار، بازار، اجرا) به‌شکل بصری ساده و مرتب. این بخش امضای برند است؛ تمیز و باکلاس باشد.

5. **فرایند همکاری (۴ مرحله)**
   - ثبت درخواست → گفت‌وگوی اولیه → جلسه‌ی مشاوره → نقشه‌ی راه. به‌صورت تایم‌لاین/مرحله‌ای.

6. **اعتبار (Credibility)**
   - آمار: ۷+ سال، ۲۰۰+ پروژه، تیم ۱۲ نفره.
   - دو نقل‌قول مشتری از brief.

7. **فرم درخواست مشاوره (مهم‌ترین بخش)** — جزئیات پایین.

8. **فوتر**
   - لوگو، اطلاعات تماس (ایمیل، تلفن، تهران)، لینک‌های لنگری، کپی‌رایت.

## مشخصات فرم درخواست مشاوره

این قلب سایت است. فیلدها:

| فیلد | نوع | الزامی |
|---|---|---|
| نام و نام خانوادگی | text | بله |
| شماره‌ی تماس | tel | بله |
| ایمیل | email | خیر |
| نام کسب‌وکار | text | بله |
| حوزه‌ی فعالیت | text | خیر |
| مرحله‌ی کسب‌وکار | select (ایده / نوپا / در حال رشد / تثبیت‌شده) | بله |
| بزرگ‌ترین چالش فعلی شما چیست؟ | textarea | بله |
| زمان مناسب برای تماس | select (صبح / بعدازظهر / عصر) | خیر |

رفتار فرم:
- اعتبارسنجی سمت کلاینت با پیام‌های فارسی واضح.
- در submit: ذخیره در جدول `leads` در Supabase.
- پیام موفقیت: «درخواست شما ثبت شد. تیم آرکان ظرف ۲۴ ساعت کاری با شما تماس می‌گیرد.»
- مدیریت حالت loading و error.
- بدون استفاده از `<form>` سنتی اگر در محیط محدود هستی؛ در Next.js معمولی استفاده از فرم و Server Action مجاز و ترجیح داده می‌شود.

اسکیمای جدول `leads` (Supabase):
```
id (uuid, pk), created_at (timestamptz),
full_name (text), phone (text), email (text),
business_name (text), industry (text),
stage (text), challenge (text), preferred_time (text),
status (text, default 'new')
```

## کیفیت و الزامات فنی

- **ریسپانسیو** کامل؛ موبایل تمام‌عرض، منوی همبرگری در موبایل.
- **دسترسی‌پذیری:** کنتراست رنگ مناسب، لیبل برای همه‌ی فیلدها، ناوبری با کیبورد، `alt` برای تصاویر.
- **SEO پایه:** متادیتای فارسی، `og:` تگ‌ها، تایتل و description مناسب، فاوآیکون.
- **پرفورمنس:** بهینه‌سازی فونت و تصویر، lazy-load، بدون کتابخانه‌ی اضافی سنگین.
- کد تمیز و کامپوننت‌محور؛ هر بخش یک کامپوننت جدا.

## ترتیب کاری که از تو می‌خواهم

1. اسکیل `/ui-ux-pro-max` را فعال کن، سپس دو فایل brief و brand-guide را بخوان و یک خلاصه‌ی کوتاه از درکت بده.
2. ساختار پروژه و `tailwind.config` با توکن‌ها را بساز.
3. لِی‌اوت پایه (فونت، RTL، رنگ‌ها) و هدر/فوتر.
4. بخش‌ها را یکی‌یکی بساز (هیرو → خدمات → چهار رکن → فرایند → اعتبار → فرم).
5. اتصال فرم به Supabase.
6. بازبینی ریسپانسیو، دسترسی‌پذیری و SEO.

در هر مرحله مختصر توضیح بده چه کردی و بعد ادامه بده.
